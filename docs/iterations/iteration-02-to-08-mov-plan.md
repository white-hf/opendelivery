# Iteration 02–08：多城市最小可运营版本执行计划

## I02：多城市站点与路由基础

**目标**：上游只提供地址和服务要求，系统把运单确定性分配到正确城市站点。交付 station 城市/省/国家字段、一城一站约束、`station_service_area`、地址标准化、`ShipmentRoutingService`、Waybill 当前路由结果、路由异常 Case、管理员站点选择器。代码决定匹配和门禁；不持久化算法中间步骤。完成标准：至少三城市的完整邮编/最长前缀/城市规则、无匹配、多匹配、人工覆盖、地址更新和跨站点拒绝 E2E；当前 `targetStationCode` 必填契约迁移为可选 hint。

## I03：运营身份、站点上下文与班前准备

**目标**：普通运营用户进入默认站点，管理员可切换站点且不串缓存。交付运营用户/角色/default station、Token、权限、审计、站点列表、班前检查和 Web 应用壳。完成标准：角色×动作矩阵、普通用户越站 403、管理员切站、TanStack Query 清理和密钥不落日志。

## I04：多城市入站闭环

**目标**：各站从路由后的 Manifest 完成实物收货。交付 Manifest 列表/详情、幂等扫描、差异 Case/决策、关闭命令、custody 和 Web Scanner。完成标准：三站并发正常、少货、多货、错站、重复扫描和并发关闭；跨站点收货失败，数量可从事实重算。

## I05：调度与装车交接

**目标**：从本站可派库存到本站司机正式持有。交付候选库存、波次草稿、发布/撤回门禁、并发锁、LOAD 会话、司机提交、主管审批和 custody 转移。完成标准：跨站 Parcel/Driver 拒绝；并发不重复分配；有差异未经审批不能出车。

## I06：失败、回站与同站重派

**目标**：未妥投包裹回到原站或有批准的下一动作。交付失败原因、重试限制、RETURN、站点接收、任务 closeout 和同站次日重派；地址变更到其他城市创建 Case，不自动转站。完成标准：领出=妥投+交回+批准持有；跨站回站、无原因失败和超限重试均拒绝。

## I07：异常与上游回传

**目标**：所有非自动场景有负责人，外部通知最终可恢复。交付逐站 Case 队列/领取/处理/关闭、SLA、Outbox/attempt/ACK 查询、死信重放和审计。完成标准：超时、5xx、业务拒绝、重启和重复重放测试；站点队列不混合，重放不重复领域状态。

## I08：逐站点日终与试运营

**目标**：每个城市站点独立解释并签署一天流转。交付逐站 reconciliation、明细钻取、结转、签字、次日待办、Runbook 和培训。用至少三个城市、每城一个站点连续运行五个营业日。完成标准：P0 缺陷为零；一个站点不平不阻断其他站点；所有不平项关联 Case；形成 `0.5` 发布/回滚决定。

## 跨迭代 Definition of Done

- 每个写 API 有权限、站点一致性、幂等、乐观锁、审计和稳定错误码。
- 每个 schema 变更有 Flyway、字段字典、索引依据和保留策略。
- 每个状态变化有合法转换、不可变事件、Outbox 和回归测试。
- 每个页面有 loading/empty/error/403/409、组件测试和 Playwright 主路径。
- 每次结束有运营演示、证据、回顾、双语总结和回滚说明；未完成项不计 Done。
