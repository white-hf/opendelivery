# R01.1 多语言基础迭代

## 目标

在继续扩大运营 Web 和司机流程前建立统一国际化底座，使三种首期语言不改变业务契约和状态逻辑。

## 垂直切片

1. 数据/API：增加账户与站点 locale，解析 `Accept-Language`，本地化通用成功、认证、权限、参数、资源和 R01 区域错误；完成司机及运营 API 测试。
2. 运营 Web：语言切换、持久化、菜单/登录/R01 区域页面翻译，缺键回退和组件测试。
3. 司机契约：输出 Android 翻译键清单和 API 行为；核心扫描、错扫、提交、派送、失败回仓文案覆盖三语。
4. 门禁：校验资源键一致、三语言回归、真实 MySQL locale 持久化、文档与测试报告同步。

## 完成标准

三种语言均可完成登录和 R01 关键流程；非法或缺失语言安全回退 `en-CA`；客户端只依赖业务码；新增文案若缺任一首期语言则 CI 失败。

## 当前状态（2026-07-20）

切片 1 已完成：Flyway V9 已应用真实 MySQL；司机/运营 locale 持久化、请求头解析、统一 API 消息和三语资源已验证。切片 2 已完成基础框架及登录、导航、区域页面切换，尚需覆盖 manifest、dispatch、case、closeout 全部文案。切片 3 Android 资源清单与切片 4 完整 UI/E2E 门禁待完成，因此 R01.1 仍为进行中。
