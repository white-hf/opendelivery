# R02.2 运营体验修复迭代

## 背景与目标

运营验收确认控制塔可以指出阻塞，但订单准备入口为空、派送规划地图和操作层级不可用，且未实现页面被当成正常菜单暴露。本迭代优先恢复可执行闭环，不扩张路线优化等范围。

| 切片 | 结果 | 验收 |
|---|---|---|
| UX01 地图可靠性 | 复用区域地图已验证的 Google Data 生命周期，点位可见、点击联动详情 | 69 个可定位 fixture、点位点击、fit、空态、Google 错误浏览器测试 |
| UX02 订单准备 | 72→66 的差异可查询、解释和处理 | 缺坐标/未分区队列、详情、重新匹配、人工区域覆盖、Case、聚合=明细 |
| UX03 规划三阶段 | 运力准备→包裹分配→预检发布 | 单一主动作、司机任务板、地图主区、按需工作列表、状态门禁 |
| UX04 导航真实性 | 不再把运营引到空白页 | 未实现阶段锁定并说明依赖；已实现配置可用 |
| UX05 配置 | 司机和站点基础资料闭环 | 司机 CRUD/停用/语言/区域偏好；站点查看与管理员受控修改 |

所有切片必须完成三语言、站点隔离、审计、单元/API/浏览器测试和中英文总结。
