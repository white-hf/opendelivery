# R02 地图规划迭代

## 背景与目标

运营必须在实物到仓前，按站点、营业日、包裹地理分布和司机当日运力建立派送批次。R02 交付可用于三城市试运营的完整计划闭环，而不是演示型单司机建单页面。

## 业务闭环

1. 已路由订单经地理编码后匹配到已发布区域版本；无法定位、未匹配区域、地址冲突进入异常层。
2. 运营选择站点和营业日，维护司机出勤、包裹容量并创建空批次草稿。
3. 地图展示区域、包裹、异常和司机任务；运营可整区分配，也可选择若干包裹拆分给不同司机。
4. 每次分配在事务中校验站点、营业日、包裹可计划状态、活动任务唯一性和司机容量。失败必须整体回滚并说明原因。
5. 冻结执行完整预检；冻结后禁止普通编辑。发布只生成司机应扫清单，不转移实物 custody。
6. 冻结前可带原因改派；系统保留原任务、目标任务、包裹、操作人和前后值审计。

## 数据与技术方案

- `driver_shift` 保存站点营业日出勤和容量快照；同一司机同日唯一。
- `dispatch_wave` 增加 `FROZEN` 状态和冻结元数据；发布必须从 `FROZEN` 进入 `PUBLISHED`。
- `driver_task_area` 保存任务引用的不可变区域版本；`driver_task_item` 仍是逐件事实和活动任务唯一约束。
- 地图查询一次返回坐标、区域、任务、司机及异常代码；详情按需加载，避免把收件人信息发给地图供应商。
- 大数据量按 `(station_id, service_date, status)`、任务状态和空间列建索引；地图按视口/游标查询，单次最多 2,000 点。

## 交付内容

- 班次/容量、批次草稿、整区/逐件分配、撤销/改派、冻结、发布 API。
- 地图优先运营工作台：包裹分布、区域、异常、司机容量、包裹抽屉和固定动作栏。
- `DEMO-R02-*` 三城市真实感数据：区域、司机班次、地址坐标、正常包裹和可解释异常；脚本可重复执行并可独立清理。
- 单元、集成和三站 E2E；同步 API、数据模型、测试与执行总结中英文文档。

## 完成标准

- YHZ、YYZ、YVR 数据和权限互不可见；跨站 ID 操作返回拒绝。
- 同一包裹不能进入两个活动任务，容量超限不能冻结，未出勤司机不能接单。
- 整区分配、拆分、改派、冻结和发布在真实 MySQL 通过；并发失败不产生部分写入。
- 发布后司机只看到本人应扫清单，包裹仍为站点 custody。
- Maven、Web 单测/类型/lint/build 和自动化 E2E 全部通过；浏览器体验验收单独记录。
