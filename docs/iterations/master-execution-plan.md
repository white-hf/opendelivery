# OpenDelivery 版本迭代主执行计划

> **已重新基线化：** 2026-07-21 起司机执行与运营能力拆为两个产品流，后续优先级、编号和联合 Gate 以 [司机产品与运营产品双轨迭代计划](two-product-execution-plan.md) 为准；本文 I07–I15 旧顺序仅作历史计划。

## 1. 目的、状态与执行假设

本文把 [版本路线图](../roadmap/version-roadmap.md) 转换为开发执行计划。I00–I01已完成；I02–I06的后端、数据库和真实 MySQL 主链已交付，但运营 Web 与系统化自动测试仍是未完成的 DoD。自 2026-07-20 起执行“试运营优先”重排：先完成三城市连续五个营业日所需的运营界面、I07、I08和自动化门禁；I09–I15全部暂停到`0.5` Gate之后。

当前关键路径以 [三城市五日试运营优先计划](pilot-first-execution-plan.md) 为准；后续版本章节仅为 Gate 后 Backlog。

日历按2026-07-20开始、一个跨职能交付单元估算；每个普通迭代5个工作日，I08为10个工作日以包含连续5天试运营。日期是基线，不是忽略质量门禁的承诺；范围变化、外部输入延迟或P0缺陷必须重新排期。

## 5. `0.5 Multi-City MOV`详细执行

### I02：多城市站点路由（2026-07-20—07-24）

**数据库**：新增station城市/省/国家、一城一站约束、`station_service_area`，Waybill增加当前`routing_status/resolved_station_id/routing_reason_code/routed_at`及查询索引。迁移包含现有数据回填和回滚说明。

**后端/API**：实现Address Normalizer、`ShipmentRoutingService`、最长邮编前缀/城市规则、唯一匹配、无匹配、多匹配、人工覆盖；将上游`targetStationCode`由必填迁移为可选hint；只有`ROUTED`创建Manifest。禁止保存算法中间候选。

**运营Web**：创建前端工程、站点选择器、服务范围列表/编辑、路由异常列表、重试和人工指定。

**测试与证据**：三城市正常路由、无匹配、多匹配、重复事件、同key异payload、人工覆盖、地址更新、站点停用；空库/存量Flyway；浏览器演示三笔运单分别进入三站。

**退出标准**：路由错误和跨站结果为零；当前E2E不再依赖上游内部站点字段。

### I03：运营身份与班前准备（2026-07-27—07-31）

**数据库**：运营用户、角色/授权或简化默认站点、会话、审计；明确普通用户固定站点、管理员可切站。

**后端/API**：运营登录/刷新/退出、`/me`、站点列表、权限拦截、班前Readiness；废弃全局Ops API Key的外部使用，保留受控迁移开关。

**运营Web**：登录、应用布局、权限路由、站点上下文、今日总览、用户管理；切站清除Query缓存。

**测试与证据**：角色×资源×动作矩阵、401/403、越站ID、Token轮换/撤销、管理员切站缓存、审计内容脱敏；用入站员/调度/主管演示各自菜单。

**退出标准**：普通用户无法通过URL或body操作其他站点，高风险动作可追溯。

### I04：多城市入站（2026-08-03—08-07）

**数据库/领域**：完善Manifest计数投影、差异Case关联和必要索引；事件仍是数量重算真值。

**后端/API**：Manifest分页/详情、开始收货、扫描、差异决策和关闭；处理正常、少货、多货、错站、破损、未知和重复；批准后原子更新库存与custody。

**运营Web**：Manifest列表/详情、扫码枪输入、即时反馈、差异抽屉、关闭门禁。

**测试与证据**：三站并行收货、设备幂等、并发关闭、跨站收货拒绝、异常Case；用脱敏实物条码样例完成一场收货演示。

**退出标准**：Manifest数量可从事实重算；所有差异已解决或有关联Case，少货不进入库存。

### I05：调度与装车交接（2026-08-10—08-14）

**后端/数据库**：候选库存、Wave草稿/明细、发布/撤回命令、短事务锁和version；验证station、routing、custody、阻断Case、任务占用和Driver本站状态。

**司机/运营Web**：司机LOAD session、扫描/提交；运营可派库存、波次详情、逐件校验、装车差异和主管审批。

**测试与证据**：两个调度并发发布、重复包裹、跨站司机/包裹、差异未批、审批事务回滚、本人任务隔离；演示一站正常和一站差异交接。

**退出标准**：同件最多一个活动任务；批准前custody不转移，批准后账实一致。

### I06：失败、回站与重派（2026-08-17—08-21）

**领域/API**：失败原因/证据规则、尝试上限、Driver Attempt V2、任务closeout、RETURN session、站点审批、同站次日重派。地址变更到其他城市创建Case，不自动改站。

**运营Web/Driver契约**：失败原因、证据提示、任务统计、回站逐件决定；旧司机API保持兼容并标记弃用路径。

**测试与证据**：成功/POD、无人、拒收、地址错误、重复幂等、超限、离线重传、跨站回站、部分同步失败；演示“领出=妥投+交回+批准仍持有”。

**退出标准**：司机没有无解释持有；失败件均有下一动作、Case或回站事实。

### I07：Case与可靠回传（2026-08-24—08-28）

**后端/API**：Case创建/领取/转派/动作/解决/关闭和SLA；Outbox/Attempt/ACK分页详情、死信重放、Partner/站点过滤和业务拒绝Case。

**运营Web**：Case队列/时间线、超时标识、接入批次、回传详情、重放高风险确认。

**测试与证据**：超时、429、5xx、永久4xx、业务拒绝、Worker崩溃/重启、租约过期、重复重放和敏感数据脱敏；演示死信恢复到ACK。

**退出标准**：非自动场景都有owner/SLA；任何回传事件可解释、恢复或带原因豁免。

### I08：日终与多城市试运营（2026-08-31—09-11）

**第一周实现**：逐站reconciliation计算、明细钻取、重算、Case关联、结转、签字、次日待办；Runbook、培训和回滚脚本。

**第二周试运营**：至少三城市各一站连续5个营业日；每日班前、接货、派车、交接、成功/失败、回传和关站；记录单量、差异、人工干预、P0/P1缺陷和回调指标。

**退出标准**：P0=0；路由/跨站错误=0；所有司机持有和日终variance可解释；回调最终成功率≥99.5%；产品、运营、工程共同签署发布或有限延期决定。

## 6. `1.0 Stable Operations`执行条件与计划

只有`0.5` Gate通过、第二个Partner资料可用、日峰值和RTO/RPO确认后启动。

### I09：Partner Adapter（2026-09-14—09-18）

实现Adapter SPI、第二Partner入站/出站映射、Partner级HMAC/密钥版本、原文隔离和重放、契约fixture和沙箱。验收要求同一Canonical领域服务同时处理两个格式，Partner差异不进入履约代码。

### I10：变更、取消和乱序（2026-09-21—09-25）

实现外部版本/事件时间、同版本冲突、地址/服务更新、取消、履约阶段门禁、派送中拦截Case和回调。验收覆盖接入前、已路由、已到站、已派单、在途和终态矩阵。

### I11：可观测性、SLO和容量（2026-09-28—10-02）

建立业务/技术指标、结构化关联日志、Trace、告警、慢查询基线、真实比例数据压测、Outbox公平性和容量模型。输出p50/p95/p99、最大验证吞吐、瓶颈、扩容阈值和正式SLO提案。

### I12：安全、恢复和发布（2026-10-05—10-09）

完成安全测试、密钥轮换、POD访问/保留、依赖扫描、数据库备份恢复、Flyway升级/失败恢复、应用回滚和生产Runbook。`1.0` Gate必须有恢复实测RTO/RPO，不接受只描述流程。

## 7. `1.1 Operational Efficiency`执行条件与计划

只有`1.0`稳定运行并形成至少两周操作耗时、积压、妥投和回传基线后启动；每项功能必须绑定改善指标。

### I13：批量运营（2026-10-12—10-16）

实现批量差异、批量波次和批量Case，统一预检、全有或全无事务、权限、结果下载和性能限制。比较单件与批量完成相同任务的时间和错误率。

### I14：配送顺序与通知（2026-10-19—10-23）

基于现有地址/时窗提供可人工调整的基础stop sequence，不承诺优化器；实现出车、到达、失败通知、模板、退订/失败和发送审计。先对一个站点启用，再按失败率推广。

### I15：SLA与运营指标（2026-10-26—10-30）

实现SLA、首次妥投率、积压、司机closeout和回传延迟看板及受控导出；指标定义可追溯到事实表。与MOV/1.0基线比较，只有实测改善才发布`1.1`。

## 8. 测试、分支和发布策略

- 每个故事从主干短分支交付，Flyway只前向新增；禁止回改已发布migration。
- 后端门禁：Maven unit/integration/API、真实MySQL E2E、空库和升级迁移、Docker构建。
- 前端门禁：lint、typecheck、Vitest/RTL、build、Chromium Playwright；版本Gate增加三浏览器smoke。
- 每次写操作测试401/403/404/409、幂等重放、version冲突和跨站点。
- 发布采用expand/migrate/contract；先迁移兼容schema，再发布应用，最后清理旧契约。
- P0立即停止发布；P1必须有owner和明确修复版本；P2可进入排序Backlog。

## 9. 进度报告与变更控制

迭代看板列为`Ready → In Progress → Review → Verified → Accepted`。周报只报告已验收结果、测试证据、遗留风险和下一关键路径。新增范围先判断是否阻断当前版本；非阻断项进入后续Backlog，不静默扩张本周目标。任何日期调整同步更新本计划、路线图和中英文版本。
