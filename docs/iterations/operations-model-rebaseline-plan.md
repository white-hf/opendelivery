# 运营模型重基线迭代计划

2026-07-20 运营评审确认标准流程是“先按订单/区域/运力规划任务，后由司机在实物到仓时扫描领取”，替代旧的“运营逐件入站后才派车”。I02–I06 的路由、身份、扫描、custody、失败和回仓基础保留，但 DoD 按本计划重验；I07–I15 旧顺序暂停。

| 迭代 | 可运营结果 | 主要任务 | 完成证据 |
|---|---|---|---|
| R01 区域基础 | GeoJSON 小区可版本发布 | Spatial schema、导入/验证、司机偏好、区域审计 | Polygon/重叠/边界/回退单测和 MySQL |
| R01.1 多语言基础 | 运营与司机接口三语一致 | locale 持久化、API 消息、Web/Android 资源、缺键门禁 | `en-CA/fr-CA/zh-CN` API/UI/回退测试 |
| R02 地图规划 | 到仓前建立批次 | 地理编码、聚类、包裹抽屉、班次/容量、整区/拆分、冻结发布 | 三站规划 E2E |
| R02.1 运营控制塔 | 一眼知道阶段、阻塞和下一步 | 流程导航、今日控制塔、指标钻取、行动队列、地图可观察性 | 三角色寻路、69/72 点位口径、三站浏览器 E2E |
| R03 到仓实物 | 卡车和板独立登记 | Inbound Trip、Handling Unit、聚合/观察、审计 | 板跨司机/任务跨板 fixture |
| D01-D04 司机产品 | 司机本人完成任务、扫描、派送和回仓 | 独立 Driver App/API 计划，不在运营 Web 实现司机动作 | 司机身份、幂等、离线和本人权限 E2E |
| O04-O05 扫描监督/交接 | 运营观察进度、处理差异并批准 custody | 只读扫描事实、例外关闭、审批事务、安全改派 | 守恒/并发/回滚/跨站测试 |
| O06 配送回仓监督 | 运营监控派送并审批回仓 | 在途/失败队列、RETURN 审批、退站后改派 | 三站责任守恒 fixture |
| R07 日终/自动化 | 逐站可计算关站 | reconciliation、钻取、Case/结转/签字、一键环境 | Maven/Vitest/Playwright/MySQL |
| R08 五日试运营 | 三城市真实运营 | Runbook、培训、值班、日报、缺陷和 Gate | 五日签字，P0=0 |

R01 执行状态（2026-07-20）：Flyway V8、区域生命周期、司机偏好和包裹空间匹配 API 已完成；运营 Web 已支持 GeoJSON 导入、地图预览/点击绘制、校验/发布，以及区域默认司机和优先级维护。`scripts/delivery-area-e2e.sh` 已在真实 MySQL 自动验证 YHZ/YYZ/YVR 各自建区和发布、共享边界允许、面积重叠拒绝、跨站资源访问拒绝及退出清理。地图浏览器视觉验收仍待具备浏览器工具的环境执行，因此 R01 暂不标为完成。

R01 验收缺陷切片：管理员站点下拉必须展示全部活动站点；导入必须支持 geojson.io 常见的 `FeatureCollection`、`Feature`、`Polygon`、`MultiPolygon`，支持选择本地 `.geojson/.json` 文件并立即回显；地图点击绘制必须持续显示节点、闭合边界、填充和节点数量。解析失败要在提交前给出明确提示。上述缺陷完成自动化回归并由运营人员浏览器复验后关闭 R01。

R01 体验验收切片：区域编辑采用视口内弹窗，地图与导入内容在宽屏并排、窄屏堆叠，提交/取消固定在弹窗底部，不允许主操作随内容滚出视野。业务操作成功或失败使用高可见度全局通知；字段校验仍就近显示。验收目标是在常用笔记本分辨率下无需滚动即可看到地图、导入入口和“创建草稿”。

R01 区域生命周期闭环：区域工作台以地图为主要视图，列表用于筛选和选择；必须支持新增、地图/版本详情查看、元数据与边界修改（生成新草稿版本）、校验、发布、安全停用及重新启用。`areaCode` 创建后不可变；“删除”是可审计的逻辑停用，不物理删除已被历史计划引用的区域或版本。停用区域不参与新包裹匹配，重新启用后恢复其已发布版本。所有写操作要求原因并记录操作人、站点、前后状态和 request ID。

运营系统采用“地图优先的 Last Mile 控制台”：地图是区域、包裹分布、司机任务和异常的主工作区，列表/详情通过侧栏或抽屉辅助。R01 地图底座切换为 Google Maps JavaScript API，密钥仅通过 `VITE_GOOGLE_MAPS_API_KEY` 注入，禁止写入仓库；必须在 Google Cloud 限制允许域名和 Maps JavaScript API，并设置配额/费用告警。区域编辑只向 Google 加载地图视图，不发送包裹或收件人业务数据。参考工具中的 `drawing` 库已不作为新实现依赖，边界点击采集由应用自行维护。

发布验证采用 `ST_Intersects` 判断点在面内（包含边界），同层发布区域用 `ST_Intersects AND NOT ST_Touches` 拒绝面积重叠但允许共享边界。每次发布保留不可变版本，规划结果引用具体版本，后续修改规则不会改写历史任务。

区域发布、本人扫描、活动任务唯一、custody、跨站拒绝和日终守恒不能人工替代。现场司机交接、自动区域建议、动态优化和高级 SLA 延期。

自 2026-07-21 起，后续编号和产品边界以 [司机产品与运营产品双轨迭代计划](two-product-execution-plan.md) 为准。运营端“司机扫描”仅表示监督工作台，不提供代司机扫描能力。
