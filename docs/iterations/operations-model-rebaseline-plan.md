# 运营模型重基线迭代计划

2026-07-20 运营评审确认标准流程是“先按订单/区域/运力规划任务，后由司机在实物到仓时扫描领取”，替代旧的“运营逐件入站后才派车”。I02–I06 的路由、身份、扫描、custody、失败和回仓基础保留，但 DoD 按本计划重验；I07–I15 旧顺序暂停。

| 迭代 | 可运营结果 | 主要任务 | 完成证据 |
|---|---|---|---|
| R01 区域基础 | GeoJSON 小区可版本发布 | Spatial schema、导入/验证、司机偏好、区域审计 | Polygon/重叠/边界/回退单测和 MySQL |
| R02 地图规划 | 到仓前建立批次 | 地理编码、聚类、包裹抽屉、班次/容量、整区/拆分、冻结发布 | 三站规划 E2E |
| R03 到仓实物 | 卡车和板独立登记 | Inbound Trip、Handling Unit、聚合/观察、审计 | 板跨司机/任务跨板 fixture |
| R04 本人扫描 | 司机只接受本人包裹 | 待扫、拒绝型错扫、破损、提交、离线幂等 | 正确/错任务/重复/未知/破损测试 |
| R05 核对/交接 | 全批次差异可解释并批准 custody | 进度、例外关闭、跨任务重算、审批事务、安全改派 | 守恒/并发/回滚/跨站测试 |

| R08 五日试运营 | 三城市真实运营 | Runbook、培训、值班、日报、缺陷和 Gate | 五日签字，P0=0 |

R01 执行状态（2026-07-20）：Flyway V8、区域/版本/偏好/地理编码/匹配归属和审计表已完成；草稿、校验、发布、列表/版本 API 已完成；运营 Web 已完成 GeoJSON 导入、验证和发布入口。MySQL 已验证 SRID 4326、有效性、空间索引、点面相交及重叠拒绝，前后端单测已通过。区域内嵌地图预览/绘制、司机偏好 API、包裹空间匹配服务和三站隔离 E2E 仍在 R01 后半段，不得把 R01 标为完成。

发布验证采用 `ST_Intersects` 判断点在面内（包含边界），同层发布区域用 `ST_Intersects AND NOT ST_Touches` 拒绝面积重叠但允许共享边界。每次发布保留不可变版本，规划结果引用具体版本，后续修改规则不会改写历史任务。

区域发布、本人扫描、活动任务唯一、custody、跨站拒绝和日终守恒不能人工替代。现场司机交接、自动区域建议、动态优化和高级 SLA 延期。
