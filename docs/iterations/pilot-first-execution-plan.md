# 三城市五日试运营优先执行计划

## 目标与边界

唯一当前目标是让 YHZ、YYZ、YVR 三个独立单站点连续五个营业日完成班前准备、接入、入站、调度、交接、妥投/失败回站、Case、回传和日终签字。第二 Partner、批量效率、通知和高级指标暂缓。

## 优先迭代

| 顺序 | 迭代 | 可运营结果 | 完成证据 |
|---|---|---|---|
| P0-A | 运营 Web 最小工作台 | 登录/切站、总览、Manifest、调度、Case、回传、日终 | Vitest + Playwright；三角色浏览器演示 |
| P0-B | I07 Case 与可靠回传 | 异常有 owner/SLA；Outbox 可查询、解释、重放 | 429/5xx/永久4xx/超时/重放自动测试 |
| P0-C | I08 日终关站 | 每站库存、司机持有、异常、回调差异可重算、结转、签字 | 三站每日 reconciliation fixture |
| P0-D | 自动化门禁 | 一条命令建立隔离数据并跑三站完整业务日 | Maven、真实 MySQL、Playwright、迁移升级 |
| P0-E | 五日执行包 | Runbook、账号、值班、回滚、日报、缺陷模板 | 连续五个营业日人工签字 |

## 2026-07-20 执行快照

- P0-A 进行中：React 19/TypeScript/Vite/Ant Design 运营端已实现登录、按角色导航、管理员切站、就绪总览、Manifest 接收/扫描/关闭、派车候选/建波次/发布、Case 列表；回传与日终页面等待 P0-B/P0-C API。
- 单元测试：Maven reactor 13 个、Vitest 8 个均通过；前端严格类型检查和生产构建通过。
- 真实 MySQL：Flyway V1-V7 校验通过；三站各 6 个运营读模型共 18 个鉴权请求全部成功。
- `scripts/db/003_three_station_pilot_seed.sql` 可重复建立三站试运营人员和基础服务区；仅限非生产，使用后必须轮换凭据。
- P0-A 未完成项：浏览器 Playwright 流程、Manifest 差异处理 UI、装车审批 UI，以及回传/日终模块。

## 测试优先级

- 单元测试优先覆盖路由选择、状态门禁、权限矩阵、计数重算、失败规则、Case 状态机、回传退避和日终不变量；新增业务逻辑必须同时新增单元测试。
- 每站 E2E 覆盖正常、少货、多货、破损、失败重派、退上游和地址 Case。
- 权限覆盖无 Token、错误角色、跨站 URL/body/header、司机操作他人任务。
- 一致性覆盖重复上游/设备事件、并发发布/关闭、Token 轮换和活动任务唯一性。
- 日终不变量：`期初 + 入站 - 妥投 - 退上游 = 站内 + 司机持有 + 已解释差异`。

## Gate

系统准备 Gate 要求单元/集成/E2E 全绿、P0=0、跨站错误=0、差异可解释、回调测试最终成功率≥99.5%。真实五日 Gate 还需要三城市运营人员、司机、数据和每日签字；模拟不能替代外部证据。
