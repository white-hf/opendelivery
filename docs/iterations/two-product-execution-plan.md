# 司机产品与运营产品双轨迭代计划

## 产品边界

系统按两个独立产品规划、验收和发布：

1. **司机 App 与司机后台 API**：服务司机本人，负责登录、本人任务、装车扫描、提交领取、派送、POD、失败处理、回仓扫描和离线同步。Android App 不在本仓库时，本仓库仍需完整交付并测试 Driver API 契约、状态机和模拟客户端 E2E。
2. **运营系统与运营 API**：服务站点运营、调度和主管，负责订单准备、区域/运力规划、到仓实物、扫描进度观察、交接审批、派送监控、异常处置和日终关站。运营端不替司机执行扫描或派送。

二者共享 `Parcel`、`DriverTask`、`ScanSession/Event`、`CustodyEvent`、`Case` 和状态机，但不得共享客户端权限。Driver API 仅从 Token 取得 driver identity；Operations API 强制角色和站点上下文。

## 司机产品迭代

| 迭代 | 可用结果 | 核心任务 | 完成证据 |
|---|---|---|---|
| D01 本人任务 | 司机只看到本人已发布任务和应扫清单 | 任务摘要/明细、任务归属、状态门禁、三语言 | 本人/他人/撤销任务 API 测试 |
| D02 本人扫描 | 司机完成领取扫描并提交 | 开启/恢复 LOAD Session；`EXPECTED/WRONG_TASK/UNKNOWN/DUPLICATE/DAMAGED`；离线 `deviceEventId` 幂等；提交快照 | 错司机不入有效列表、不改变 custody；重复事件计数不变 |
| D03 派送执行 | 司机完成妥投或失败记录 | 出发、任务列表、POD、失败原因、再次尝试、离线同步 | 本人权限、POD 完整性、幂等及失败状态机测试 |
| D04 回仓交回 | 未妥投件逐件交回 | RETURN Session、扫描、提交、司机 closeout | 未交回件可定位；提交不直接把 custody 转回站点 |

## 运营产品迭代

| 迭代 | 可用结果 | 核心任务 | 完成证据 |
|---|---|---|---|
| O03 到仓关联收尾 | Trip/板与上游包裹及任务自动关联 | Unit 标识接入、跨司机覆盖、三城市 fixture | 板跨司机、任务跨板、聚合=明细 |
| O04 扫描监督 | 运营查看司机扫描进度，不代替扫描 | 按批次/任务/司机展示应扫、有效、漏扫、错扫、破损、未提交；异常钻取 | 与 Driver Scan Event 同源；跨站隔离 |
| O05 交接审批 | 主管批准责任交接 | 全批次门禁、差异决定、拒绝重开、安全改派、原子 custody 转移 | 并发审批唯一成功；无有效扫描不得放行 |
| O06 派送与回仓监督 | 运营跟踪在途、失败和退站 | 地图/列表监控、失败队列、RETURN 审批、重派 | 包裹状态与 custody 守恒 |
| O07 日终关站 | 每站独立核对并签字 | 库存、司机持有、任务、Case、回调差异、结转 | 不满足硬门禁无法关站；结果可重算 |
| O08 基础配置与异常中心 | 试运营主数据和补偿闭环 | 司机/站点配置、Case owner/SLA、Outbox 查询重放 | 审计、RBAC、重试/死信测试 |

## 联合发布 Gate

联合 E2E 串联：上游订单 → 运营规划 → 到仓 → **司机 D02 扫描提交** → **运营 O05 审批** → 司机 D03 派送/D04 回仓 → 运营 O06 审批/O07 关站。任何一个产品通过自身测试但未通过跨产品契约与责任守恒，都不能进入三城市五日试运营。

优先依赖链为 `O03 → D01 → D02 → O04 → O05 → D03 → D04 → O06 → O07 → 联合 E2E`。O04 可在 D02 API 稳定后并行开发；运营端不再建设司机扫码输入界面。
