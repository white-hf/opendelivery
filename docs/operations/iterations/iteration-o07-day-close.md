# O07 日终关站迭代

> 状态：`COMPLETED`（2026-07-21 完成）；归属：运营产品；对应双产品计划 O07，是 0.5 MOV 试运营每日闭环。已关联 [执行总结](../summaries/iteration-o07-day-close-summary.md)。

## 可运营结果

每站每日独立核对并签字关站：库存、司机持有、任务闭环、Case、回调差异全部可解释；不满足硬门禁无法签字；签字结果可重算、签字后只读。

## 范围

1. **核对计算**：`POST /ops/v1/day-close/recalculate {serviceDate}` 按站计算并写 `daily_reconciliation`（未签字可重算覆盖）：站点库存件数（STATION custody）、司机持有（DRIVER custody 按人）、任务闭环（CLOSED 与未闭环清单）、未闭环 LOAD 审批、未决 Case、Outbox 死信/未 ACK、Manifest 未关差异。
2. **硬门禁**：司机无未解释持有（守恒公式：领出=妥投+交回+批准持有）、无未闭环审批、无未决 P0 Case、死信=0 或全部挂 Case；门禁逐项列出可执行处理入口。
3. **签字**：`POST /ops/v1/day-close/sign` 主管签字（记录签字人、时间、计算快照）；签字后该日核对只读，纠错通过结转或审计动作，不覆盖历史。
4. **结转**：门禁未过项可逐项填原因并关联 Case 后结转次日待办；结转留审计。
5. **钻取**：每个指标可钻到明细（复用 O04 扫描、O06 在途/失败、Case、Outbox 查询）。
6. **前端**：日终页（指标卡+门禁状态+钻取+重算+签字+结转），三语言；导航"日终关站"进入。Runbook 增补关站操作条目。

## 非目标

- 不做跨站/总部汇总与多时区扩展（站时区已生效）；不做自动结转（人工确认）；不做财务口径对账。

## 依赖

- O05 custody 语义、O04/O06 口径、Case/Outbox 主链、`daily_reconciliation`（V1）；web-spec 4.9 旧版页面规格作底稿。

## 迁移与兼容

- `daily_reconciliation` 如需扩展列（签字快照/门禁明细 JSON），新增 Flyway（先取号）；只增不改。

## 风险

- 重算与签字并发：站点+日期行悲观锁，签字后禁止重算（409）。
- 指标口径与 O04/O06 不一致 → 统一走同源查询服务，禁止另写口径。

## 测试与 DoD

- Java：重算幂等、门禁矩阵、签字后只读、锁冲突。
- 真实 MySQL E2E：造全天数据（接入→规划→扫描→审批→派送/失败→回调）→核对→门禁拦截→处理→签字→守恒断言；自清。
- Web：Vitest + Playwright；契约、web-spec（4.9 刷新）、Runbook 中英同步。
