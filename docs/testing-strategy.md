# 测试策略

## 分层

- **单元测试**：状态规则、JWT、密码、幂等和纯映射逻辑；不依赖网络或数据库。
- **应用测试**：使用 `memory` profile 和 MockMvc 验证认证、响应契约与司机数据隔离。
- **MySQL E2E**：在真实 MySQL 上执行上游接入、到站、派车、扫描、审核、妥投，验证领域事件和 outbox，并自动清理测试数据。
- **迁移测试**：空临时 schema 执行 Flyway，核对表、外键、索引和 schema history。
- **回归测试**：覆盖现有 Android API 的字段、业务码和 multipart 行为。

## 必测失败路径

重复上游事件、未知条码、重复/串任务扫描、未授权跨司机访问、非法状态转换、缺失 POD、数据库/回调失败、outbox 重试与死信、manifest 差异、日终不平。

## 发布门禁

所有自动化测试为零失败；MySQL E2E 有持久化证据；migration dry-run 成功；不存在明文秘密；回滚和数据备份步骤经过评审。未自动化的场景必须在测试报告列出 owner 和风险。
